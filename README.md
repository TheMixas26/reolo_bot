# Предложка Империи

Данный бот разработан мной для продвинутого ведения моего канала —
[Имперский Вестник](https://t.me/reolo_channel).

Это не просто «бот для предложки». Со временем он разросся в довольно странный гибрид из нескольких систем: модерации постов, мини-банка, RPG, достижений, календаря и прочих штук, которые появились просто потому что *почему бы и нет*.

Вот основной функционал, который он предоставляет.

---

## Основной функционал «Предложки»

### Отправка постов в канал

Пользователь может отправить боту практически что угодно:

* текст
* фото
* видео
* документы
* аудио
* голосовые сообщения
* стикеры
* посты из других каналов
* **альбомы (media groups)**

Бот автоматически оформит пост, подпишет имя отправителя и отправит его **на проверку админу**.

После этого у администратора есть две кнопки:

* **Одобрить** — пост публикуется в канале
* **Запретить** — пост исчезает в небытие

Никаких лишних сложностей. Простая и удобная модерация.

---

### Поддержка хештегов

Предложка понимает несколько специальных хештегов:

* **#аноним** — скрывает имя автора.
  Причём анонимность настоящая — даже админ не увидит отправителя.

* **#вопрос** — изменяет оформление поста, превращая его в вопрос для читателей канала. (WIP, пока что выглядит странновато)

* **#ai** — вызывает встроенный ИИ (YandexGPT), который может ответить на сообщение.

### Система достижений

Пользователи могут получать **достижения**.

Они могут выдаваться:

* автоматически (например за первый пост),
* вручную администратором,
* за участие в ивентах (технически, тоже лапками админа).

Пользователь может посмотреть их через команды бота.

Есть даже **таблица достижений**, потому что людям нравится, когда у них есть пиксельная медалька.

---

### Система дней рождения

Предложка умеет:

* хранить дни рождения участников,
* уведомлять чат о приближении даты,
* отправлять **персональные уведомления** пользователям,
* поздравлять именинников.

Можно добавить день рождения себе или другому пользователю, если вы админ.

---

### Прогноз погоды

Бот может (и будет, попробуйте остановите) автоматически отправлять **ежедневный прогноз погоды** в чат обсуждения канала.

Используется API open-meteo.

Показывается:

* температура
* ветер
* условная иконка погоды

Если API внезапно решит умереть — админ получит уведомление.

### Банковская система (WIP)

Есть отдельный **банк-бот**.

Он позволяет:

* смотреть баланс
* переводить средства другим пользователям
* смотреть курс валюты

Баланс хранится в базе данных, переводы происходят с комиссией (см. `commission` в `config.py`).

Система простая и сделана больше ради фана, чем ради экономики.

---

### Карточная мини-игра (WIP)

Есть ещё один отдельный бот — **RPG-бот**.

Он позволяет:

* открывать паки через `/roll`
* собирать коллекцию карт
* запускать дуэли `/duel`
* устраивать командные бои `/team_battle`

Это простая карточная система, добавленная в проект ради фана и экспериментов.

## Архитектура проекта

Проект написан на **Python** и использует:

* `pyTelegramBotAPI (telebot)`
* адаптеры платформ для публикации (`TelegramAdapter`, `VKAdapter`)
* **SQLite** для хранения данных

Структура проекта выглядит примерно так:

```
handlers/                    обработчики Telegram-команд
handlers/card_handlers/      Telegram-слой карточного бота
posting/                     общие модели постов, сервисы и адаптеры платформ
card_game/                   доменная логика карточной игры
database/sqlite_db.py        работа с SQLite
utils/                       вспомогательные функции
achievements/                система достижений
ai/                          интеграция с YandexGPT

main.py                      точка входа
```

---

## База данных

Используется SQLite база:

```
database/bot.sqlite3
```

Основные таблицы:

* `user_accounts` — пользователи и их баланс
* `birthdays` — дни рождения
* `rpg_players` — данные RPG игроков
* `achievements` — список всех существующих достижений
* `user_achievements` — полученные достижения

Таблицы создаются автоматически при первом запуске. Не удивительно, не так ли?)

## Автоматические задачи

Бот использует `APScheduler`, чтобы запускать различные фоновые задачи:

* уведомления о днях рождения
* персональные поздравления
* прогноз погоды
* резервные копии базы данных
* автоматическая проверка достижений

## config.py

Файл `config.py` содержит **все токены и настройки** бота.

В репозитории он отсутствует - по очевидным причинам.

Если вы вдруг решите запустить бота у себя, файл должен выглядеть примерно так:

```python
import telebot
from utils.imperial_сalender import ImperialCalendar

# Дебаг режим переключает токен и id канала (и ещё некоторые штучки), чтобы мне, как разработчику было удобно
# тестировать всё на другом боте, а не на рабочем. Если вы не планируете дополнять код, ставтье False и наслаждайтесь
DEBUG_MODE = True|False

BANK_TOKEN='' # токен бота, в котором будет банковская система
RPG_TOKEN='' # токен бота, в котором будет RPG система

if DEBUG_MODE:
    channel=-1009876543210 # id тестового канала
    PREDLOJKA_TOKEN='' # <- токен тестового бота, чтобы не использовать рабочего бота для эксперементов.
else:
    channel=-100223456789 # Основной бот и основной токен соотвесвтенно
    PREDLOJKA_TOKEN='' # <- токен бота


admin=123456789 # id админа
channel_red=223456789 # id канала с припиской
chat_mishas_den=-100223456789 # id группы обсуждения с припиской '-100', название переменной обусловлено названием чата, простите
backup_chat=123123123123 # id группы, куда будут отправляться backup файлы.

commission=0.02 # коммисия при переводах
location=(55.75, 37.62)

predlojka_bot = telebot.TeleBot(PREDLOJKA_TOKEN)
bank_bot = telebot.TeleBot(BANK_TOKEN)
rpg_bot = telebot.TeleBot(RPG_TOKEN)

VK_TOKEN = ""          # токен сообщества VK
VK_GROUP_ID = None     # числовой id сообщества, например 123456789
VK_OWNER_ID = None     # например -123456789 для стены сообщества
VK_API_VERSION = "5.199"

calendar = ImperialCalendar('utils/imperial_date_generator.js')

CATALOG_ID = "{id вашего облака в Yandex.Cloud}"
SECRET_KEY = "{api ключ для работы с YandexGPT}"
```

## Запуск

Запускается всё максимально просто:

```
python main.py
```

После запуска:

* поднимается планировщик задач
* запускается основной бот
* при отключённом DEBUG-режиме запускаются также банк и RPG бот
